## Reporting Issues

Found a bug or have a feature request? Here's how to report it:

<Steps>
  <Step title="Search existing issues">
    Before creating a new issue, search the [GitHub Issues](https://github.com/ollm/OpenComic/issues) page to see if someone else has already reported the same problem or requested the same feature.
  </Step>
  
  <Step title="Gather information">
    Collect relevant information about your issue:
    - OpenComic version (Help → About)
    - Operating system and version
    - Steps to reproduce the problem
    - Screenshots or error messages
    - Log files (if applicable)
  </Step>
  
  <Step title="Create an issue">
    Go to the [GitHub Issues](https://github.com/ollm/OpenComic/issues/new) page and create a new issue. Provide a clear title and detailed description with all the information you gathered.
  </Step>
  
  <Step title="Follow up">
    Monitor your issue for responses from the developers or community. Provide additional information if requested.
  </Step>
</Steps>

<Note>
For detailed guidance on reporting bugs, see the [Bug Reports guide](/community/bug-reports).
</Note>

## Support Options by Type

### For Users

<Accordion title="I need help using OpenComic">
  - Start with the [documentation](/introduction) and [FAQ](/resources/faq)
  - Check the [troubleshooting guide](/guides/troubleshooting) for common issues
  - Search [GitHub Issues](https://github.com/ollm/OpenComic/issues) for similar problems
  - If you can't find a solution, create a new issue on GitHub
</Accordion>

<Accordion title="I found a bug">
  - Follow the [bug reporting guide](/community/bug-reports)
  - Create a detailed issue on [GitHub Issues](https://github.com/ollm/OpenComic/issues/new)
  - Include steps to reproduce, screenshots, and system information
</Accordion>

<Accordion title="I have a feature request">
  - Check if the feature has already been requested on [GitHub Issues](https://github.com/ollm/OpenComic/issues)
  - If not, create a new issue with:
    - Clear description of the feature
    - Use cases and examples
    - Why this feature would be valuable
</Accordion>

### For Developers

<Accordion title="I want to contribute code">
  - Read the [contributing guide](/community/contributing)
  - Check the [CONTRIBUTING.md](https://github.com/ollm/OpenComic/blob/master/CONTRIBUTING.md) on GitHub
  - Look for issues labeled "good first issue" or "help wanted"
  - Fork the repository and submit a pull request
</Accordion>

<Accordion title="I want to help with translations">
  - Follow the [translation guide](/community/translations)
  - Check the [TRANSLATE.md](https://github.com/ollm/OpenComic/blob/master/TRANSLATE.md) on GitHub
  - Submit your translations via pull request
</Accordion>

<Accordion title="I want to build from source">
  - Follow the [building from source guide](/guides/building-from-source)
  - Check the README.md for development setup
  - Use the development scripts for testing
</Accordion>

## Response Times

<Info>
OpenComic is maintained by volunteers in their free time. Response times may vary:
- **Bug reports**: Usually reviewed within a few days
- **Feature requests**: May take longer to evaluate and implement
- **Pull requests**: Typically reviewed within a week

Complex issues or features may require more time for proper investigation and implementation.
</Info>
